Getting Started with Scripting in th.software

Introduction

th.software supports scripting using the AngelScript engine. With scripting, you can automate tasks, customize behaviors, and extend the functionality of the app.

About AngelScript

AngelScript is an open-source, cross-platform scripting language designed to be easy to learn and use. It was created by Andreas Jonsson in 2003, and has since gained popularity in the gaming industry for its performance and flexibility.

Features

AngelScript supports many features that make it a powerful scripting language, including:

  • Object-oriented programming (OOP): Classes, inheritance, polymorphism, etc.
  • Garbage collection: Automatic memory management.
  • Function overloading: Multiple functions with the same name, but different parameters.
  • Function pointers: The ability to pass functions as arguments to other functions.
  • Exception handling: Catching and handling errors at runtime.
  • Operator overloading: Custom operators for user-defined types.

Syntax

AngelScript's syntax is similar to C/C++, which makes it easy for developers with experience in those languages to learn. Here are some basic examples:

Variables

// Declare and initialize a variable
int x = 10;

// Declare a constant
const float PI = 3.14159;

Functions

// Function declaration
int add(int a, int b)
{
    return a + b;
}

// Function call
int result = add(3, 4);

Classes

// Class declaration
class MyClass
{
public:
    // Constructor
    MyClass(int value)
    {
        m_value = value;
    }

    // Member function
    void printValue()
    {
        console::print("Value: " + formatInt(m_value));
    }

private:
    int m_value;
};

// Create an instance of MyClass
auto myObj = MyClass(42);

// Call a member function
myObj.printValue();

Documentation

The official AngelScript website provides extensive documentation, including a language reference, tutorials, and examples. You can find it at https://www.angelcode.com/angelscript/sdk/docs/manual/index.html.

Scripting Basics

Scripts can be executed in th.software's GUI, in the Config tab -> Scripting subtab. Load the script by clicking the run button on the .as or .ab file. If the script is not already compiled, you can compile it by clicking the compile button.

Script File Extensions

The scripting engine supports two file extensions:

  • .as: This is the source code file extension. Scripts written in this format can be loaded and compiled at runtime.
  • .ab: This is the compiled script file extension. Scripts written in this format are already compiled and can be loaded directly. Source code cannot be read from compiled scripts as they contain byte-code.

Script Locations

Default script directory is randomized for each software user. The default directory may be seen and/or changed in the Script directory text input.

Script Functions

Each script must contain two functions:

  • void main(): This function is called when the script is first loaded. You can use this function to initialize variables, register functions, and perform other setup tasks. For example, you can print a message to the console with console::print("Hello World!");.
  • void on_gui(): This function is called repeatedly, typically every frame. You can use this function to update the app's user interface or perform other tasks that need to be performed continuously.

There are also optional callbacks:

  • void on_new_player(player &in instance): This function is called when a new player is discovered. You can use this function to cache certain data you require anywhere else in the script.
  • void on_unload(): This function is called when the script is unloaded, and/or when the software exits. You can use this function to properly dispose of allocated resources (e.g. GUI items) or save configuration items.

Writing Your First Script

To get started with scripting in th.software, follow these steps:

  • Create a new file with the extension .as in the default script directory.
  • Write the following code in the file:
void main()
{
    // Add initialization code here
    console::print("Hello World!");
}

void on_gui()
{
    // Add GUI update code here
}
  • Save the file.
  • Open Config tab -> Scripting subtab and click on the run icon.

If everything is working correctly, you should see Hello World! printed to the console output. The on_gui() function will be called every frame and you can update the app's user interface from there.

Congratulations! You have written and executed your first script in th.software.

Conclusion

This concludes the "Getting Started" guide for scripting in th.software using AngelScript. With this knowledge, you can start writing your own scripts to automate tasks, customize behaviors, and extend the functionality of the app.
