namespace gui

Remarks

  • Please refer to the example scripts for a better understanding of how to handle GUI bindings.
  • All add_* functions, with the exception of add_subtab, return a handle to the control. You MUST properly dispose of the created controls by calling destroy_control on ALL of the handles; this can be handled in the on_unload callback.
  • Control creation functions expect a global variable declaration as the last argument. Variable must already be declared in code; it will not be created by the binding.
  • Default tab names are:
    • players
    • loot
    • visuals
    • misc
    • aim
    • customize
    • config
